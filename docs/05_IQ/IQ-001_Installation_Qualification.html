IQ-001 — Installation Qualification Protocol (설치 적격성 평가)

시스템명: PharmCam — 검체·시료 사진 기록 시스템
문서 ID: IQ-001
버전: 0.1 (Draft)
작성일: 2026-06-09
상위 문서: VP-001, RA-001 v1.3, URS-001 v1.4, DQ-001 v1.4
추적성: docs/traceability_matrix.md

이 문서가 하는 일: PharmCam이 운영 PC에 올바르게·통제된 상태로 설치되었는지 확인하기 위한 IQ 점검 항목을 정의한다.
IQ는 "설치 환경·구성요소·접근통제가 설계(DQ)대로 갖춰졌는가"를 본다(기능 동작은 OQ에서 본다).
본 문서는 초안이며, 실제 실행 전 QA/System Owner 검토·승인 후 대상 PC에서 수행한다.

1. 목적 (Purpose)

PharmCam 설치본이 검증된 버전 그대로, 권한이 통제된 위치·계정·구성으로 설치되었음을 확인한다. 특히 설치 위치·접근통제(URS-053/D-15), 무결성 코어 구성, 신뢰 시각·키 저장소 등 OQ 기능시험의 전제가 되는 환경을 점검한다.

2. 범위 (Scope)

2.1 포함

  • 설치본 진본성·버전·체크섬
  • 설치 위치(Program Files, per-machine)와 실행파일 변경 권한
  • 운영자(작업자) Windows 계정 권한(표준 사용자)
  • 데이터 저장 위치·쓰기권한·접근통제(ACL)
  • 무결성 코어 설정 잠금, 키 저장소(safeStorage/DPAPI), 감사추적 해시체인 기준선
  • 초기 계정·강제 비밀번호 변경, 설정 명세(configuration spec) 스냅샷
  • 폐쇄망(외부 호출·자동 업데이트 없음) 확인

2.2 제외

  • 기능 동작 시험(OQ-001에서 수행)
  • 반복 운영 시나리오(PQ)
  • 카메라 장치 계측 검교정

3. 시험 전제조건 (Prerequisites)

항목 기준
대상 PC Windows 10/11 단독 PC, 폐쇄망
설치 권한 설치는 IT/시스템 관리자(Windows 관리자) 가 수행
설치본 QA가 승인한 검증 빌드(버전·SHA-256 기록 보유)
배포 매체 USB/내부 공유(폐쇄망), 설치 전 체크섬 대조
자가점검 앱의 "환경 점검" 탭(관리자) 사용 가능

참고: 현 프로토타입 단계의 적용 범위·보류 항목은 프로젝트 메모리의 "배포 정책(프로토타입 단계)" 결정을 따른다(코드서명·암호화 보류, 체크섬·표준사용자·ACL 적용).

4. IQ 점검 항목 (IQ Test Cases)

판정: 적합(Pass) / 부적합(Fail) / 해당없음(N/A). 부적합은 §6 일탈로 기록한다.

IQ-TC 점검 항목 방법 기대 결과 추적(URS/DQ) 결과·판정
IQ-TC-001 설치본 진본성·체크섬 배포된 설치본의 SHA-256을 QA 릴리스 기록과 대조 해시 일치(검증 빌드와 동일) 배포정책, SEC-06
IQ-TC-002 앱 버전 일치 설치 후 "환경 점검"의 앱 버전 확인 검증된 버전과 동일(예: 0.1.0) VP, URS-073
IQ-TC-003 설치 위치(Program Files) 설치 경로가 C:\Program Files\PharmCam(per-machine)인지 확인 관리자 전용 위치에 설치됨 URS-053, D-15
IQ-TC-004 프로그램 변경 권한 표준 사용자 계정으로 설치 폴더의 실행파일 수정·삭제 시도 권한 거부(표준 사용자는 변경 불가) URS-053, D-15
IQ-TC-005 작업자 계정 권한 작업자용 Windows 계정이 표준 사용자(로컬 관리자 아님)인지 확인 표준 사용자임 URS-053, D-15
IQ-TC-006 데이터 저장 위치 "환경 점검"의 데이터 폴더 경로 확인(설정 storage.root 반영) 의도한 데이터 폴더 경로 표시 URS-047, D-11/D-15
IQ-TC-007 데이터 폴더 쓰기 권한 "환경 점검"의 "데이터 폴더 쓰기 권한" 항목 적합(쓰기 가능) URS-045, D-9
IQ-TC-008 데이터 폴더 접근통제(ACL) 데이터 폴더에 작업자 계정의 직접 수정·삭제를 제한하는 ACL 적용 확인 작업자 직접 변경·삭제 제한됨 URS-053, D-15
IQ-TC-009 키 저장소(safeStorage/DPAPI) "환경 점검"의 safeStorage 항목 사용 가능(암호화 모듈 사용 시 필수) URS-050, D-4
IQ-TC-010 무결성 코어 잠금 "환경 점검"의 "무결성 코어 잠금" 항목 audit/integrity/auth/timestamp 4개 모두 core-locked URS-072, D-7
IQ-TC-011 감사추적 해시체인 기준선 "환경 점검"의 verifyChain 항목 통과(기준선 무결) URS-040/041, D-1
IQ-TC-012 초기 계정·비밀번호 강제 변경 초기 admin 로그인 → 강제 변경 동작 확인 최초 로그인 시 비밀번호 변경 강제 URS-010, D-5
IQ-TC-013 설정 명세 스냅샷 "설정" 화면에서 configuration spec 조회·기록(IQ 첨부) 현재 설정 상태가 조회·출력됨 URS-073, D-7
IQ-TC-014 폐쇄망·자동 업데이트 없음 외부 네트워크 호출/자동 업데이트가 없음을 확인 외부 통신·자동 업데이트 없음 URS-052, DEV-04
IQ-TC-015 런타임 의존성 번들 대상 PC에 Node/빌드도구 미설치 상태에서 정상 실행 추가 설치 없이 실행됨(자체 번들) VP 범위

5. 수용 기준 (Acceptance Criteria)

  • IQ-TC 전 항목이 적합(Pass) 또는 정당한 N/A일 것.
  • 부적합 항목이 있으면 §6 일탈로 기록하고, 조치·재시험 후 합격해야 OQ로 진행한다.
  • "환경 점검" 자가점검 결과(JSON 내보내기)와 설정 명세 스냅샷을 IQ 기록에 첨부한다.

6. 일탈 처리 (Deviation)

일탈 ID IQ-TC 내용 조치 재시험 결과

7. 승인 (Approval)

역할 성명 서명 일자
수행자 (Installer/IT) (지정)
System Owner (지정)
QA (지정)

부록 — 환경 점검 탭과의 매핑

앱의 "환경 점검"(관리자) 자가점검은 다음 IQ 항목을 자동 수행·근거 제공한다: IQ-TC-007(쓰기권한), IQ-TC-009(safeStorage), IQ-TC-010(코어 잠금), IQ-TC-011(verifyChain), IQ-TC-002(버전)·IQ-TC-006(데이터 폴더). 결과는 JSON으로 내보내 IQ 기록에 첨부한다.
나머지(설치 위치·계정 권한·ACL·체크섬·폐쇄망)는 IT/관리자가 OS 레벨에서 수동 확인한다.
