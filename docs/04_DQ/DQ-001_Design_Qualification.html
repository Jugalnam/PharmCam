DQ-001 — Design Qualification (설계 적격성 평가)

시스템명: PharmCam — 검체·시료 사진 기록 시스템
문서 ID: DQ-001
버전: 1.4 (Approved — 2026-06-09 확정: D-15 설치·데이터 접근통제 설계)
작성일: 2026-06-05
확정일: 2026-06-05 (v1.0) · 2026-06-08 (v1.1·v1.2) · 2026-06-09 (v1.3·v1.4)
상위 문서: URS-001 (요구사항), URS-002 (적합성·일탈)

이 문서가 하는 일: "무엇을 해야 하는가(URS)"를 "어떻게 만들 것인가(설계)"로 옮기고,
그 설계가 URS 46개를 만족함을 문서로 검증한다(design verification).
URS-002의 일탈(DEV) 처리 중 설계로 해결할 부분(해시 체이닝·키 관리 등)도 여기서 구체화한다.

1. 목적·범위

URS의 모든 요구사항을 충족하는 시스템 설계를 정의하고, 설계가 요구사항·위험통제에 적합한지 확인한다. 본 DQ는 상세 구현(코드) 전 단계의 설계 명세 + 설계 검증이다.

4. 핵심 설계 결정 (URS·일탈 대응)

# 설계 결정 충족 URS / 일탈
D-1 감사추적 = append-only + 해시 체이닝: 각 엔트리에 자신의 해시와 직전 해시(prevHash)를 저장해 사슬을 만든다. 중간을 고치면 사슬이 깨져 변조가 탐지된다. 편집·삭제 API 미제공 URS-040·041, DEV-01
D-2 원본 이미지 보호: 저장 시 SHA-256 계산·기록, 파일 읽기전용 속성, 조회 시 해시 재검증 URS-043, DEV-01
D-3 시각 서비스: OS 시계 사용 + 직전 레코드 시각과 비교해 역행·점프를 탐지·로깅. 네트워크 가능 시 NTP 동기화 보조 URS-044, DEV-02
D-4 암호화 + 키 관리: 선택 시 DB·이미지 암호화, 키는 OS 보호 저장소(Windows DPAPI)에 보관. 키 관리 방식은 회사 보안정책에 맞춰 IQ에서 확정 URS-050, DEV-03
D-5 인증·권한: 로컬 계정, argon2 해시, RBAC(역할별 권한), 세션 타임아웃(설정값), 로그인 실패 잠금 URS-010~016·051
D-6 전자서명: 서명 시 비밀번호 재인증 → 서명 레코드(서명자·의미·시각·sigHash)를 대상 기록에 연결. 위조·재사용 방지 URS-020~023
D-7 설정 관리: 설정 항목에 coreLocked 플래그. 코어(감사추적·식별·원본보호·시각·필수항목)는 coreLocked=true로 변경 불가. 모든 설정 변경은 AuditEntry로 기록. 현재 설정 조회·출력 기능 URS-070~073, 설정우회 위험
D-8 백업 모듈: mode(internal/external/off), 내장 사용 시 백업본 해시로 무결성 검증. 외부 모드는 회사 자체 백업 경로 지정 URS-060~062, DEV-05
D-9 저장 파이프라인: 촬영→미리보기→메타데이터 입력(필수항목 검증)→이미지+메타데이터 원자적(atomic) 저장→저장 성공 검증. 실패·재촬영도 기록 URS-030·032·033·045
D-10 LIMS 커넥터: 표준 인터페이스(확장 지점)만 정의, 기본 비활성. 실제 어댑터는 후속 변경관리 URS-080·081, DEV-07
D-11 저장 위치 설정: 원본 이미지·데이터 저장 경로를 config(storage.root, coreLocked=false)로 분리. 기본값 userData/data 유지. 변경은 관리자만, 경로 유효성·로컬 여부·쓰기권한 검증(네트워크/UNC 거부) 후 AuditEntry로 이전→이후 경로 기록. 마이그레이션 미수행 — 기록별 절대경로(imageRef) 보존으로 기존 데이터는 옛 위치에서 계속 조회·해시검증되고 신규 저장만 새 경로 사용. 위치와 무관하게 무결성 코어(D-1·D-2)는 강제. UI에 현재 저장 위치 표시·폴더 열기 제공 URS-047, DAT-08
D-12 통제 인쇄(Controlled Printout): 전자기록을 원본으로 유지하되, 종이 사용이 필요한 경우 PharmCam 내 인쇄 화면에서만 공식 출력본을 생성한다. 인쇄 미리보기에는 User ID, 시험번호, 촬영일시, 기록 ID, 파일명을 표시하고 사용자가 확인 후 인쇄한다. 인쇄 요청은 Main 프로세스에서 처리하며, 성공/실패와 관계없이 print 또는 print_failed 감사추적을 남긴다. 출력물에는 "PharmCam Controlled Printout" 및 "저장 폴더 이미지 직접 인쇄본은 공식 출력본 아님" 문구를 포함한다. OS/외부 프로그램으로 이미지 파일을 직접 인쇄하는 행위는 앱이 원천 차단할 수 없으므로 SOP와 화면 문구로 공식성 판단을 통제한다 URS-093, DAT-09
D-13 기록 조회 통제(Record Query Control): 기록 목록은 기본적으로 오늘 날짜 범위로 조회한다. 날짜/기간 필터는 모든 역할에 제공하되, operator는 Main 프로세스 쿼리에서 operator_id = currentUser.id 조건을 강제하여 본인 기록만 조회·상세조회할 수 있게 한다. reviewer/admin은 전체 기록 조회와 사용자별 필터(operatorId)를 사용할 수 있다. Renderer의 필터 표시 여부와 관계없이 record:list, record:get, 통제 인쇄 미리보기/인쇄는 Main에서 동일 조회 권한을 적용한다 URS-094·095, DAT-10, AUTH-02
D-14 촬영 카메라 선택/전환: 촬영 화면에서 navigator.mediaDevices.enumerateDevices()로 연결된 비디오 입력 장치를 열거하고, 사용자가 드롭다운 선택 또는 "카메라 전환" 버튼으로 장치를 순환 선택한다(전면↔후면은 장치 2개로 처리). 선택 시 기존 스트림을 정지하고 해당 deviceId로 재취득한다. 기본값은 후면(facingMode:environment) 우선. 카메라 선택은 무결성과 무관(메타데이터 각인·해시·감사추적 불변) URS-035, SW-03
D-15 설치·데이터 접근통제(다층 방어): ① 프로그램은 C:\Program Files에 per-machine 설치(표준 사용자가 실행파일 변경 불가) ② 작업자 Windows 계정=표준 사용자(로컬 admin 금지) ③ 데이터 폴더 NTFS ACL로 작업자 직접 수정·삭제 제한 ④ 암호화(at rest, 선택 모듈) ⑤ 해시체인 verifyChain·이미지 읽기전용으로 탐지 ⑥ 접근통제 SOP·정기 무결성 검토. 단독 PC·동일 사용자 토큰 특성상 100% 예방은 불가 → 예방+탐지+절차 다층으로 잔여 위험 관리, IQ에서 적용 확인 URS-053, SEC-06

6. 기존 솔루션 평가 (Solution Assessment)

GAMP 5는 맞춤 개발에 앞서 기존 솔루션 활용 가능성을 평가하도록 권고한다. 본 절은 그 평가 기록이다.
(평가 일자 2026-06-05, GitHub/오픈소스 조사 기반)

6.1 평가 대상 및 부적합 사유

솔루션 성격 부적합 사유
eLabFTW 서버 기반 오픈소스 ELN 폐쇄망·단독 PC 카메라 캡처 요건과 불일치, 촬영 워크플로우 부재
SciNote 서버 기반 오픈소스 ELN 동일 (서버·웹 전제)
OpenSpecimen 서버 기반 오픈소스 biobank LIMS 검체 관리 중심, 촬영 기록 앱 아님, 서버 전제
attest / Trillian 등 감사추적 외부 앵커링·네트워크 전제 폐쇄망(외부 네트워크 차단)에서 사용 불가

6.2 평가된 패턴의 차용 여부

패턴 판단 채택
RFC3161 신뢰 타임스탬프 / 외부 앵커링 폐쇄망에서 외부 서비스 접근 불가 ❌ 미채택
HMAC 키 기반 해시체인 오프라인 동작 가능하나, 현 URS(변조 탐지)는 평문 SHA-256 해시체인으로 이미 충족 → 과잉 ❌ 미채택
전자 witnessing / LIMS REST 커넥터 현 범위 외 후속(설정 옵션·변경관리)으로 보류

6.3 평가 결론

  • 폐쇄망·단독 PC·카메라 캡처 요건을 충족하는 기성/오픈소스 솔루션이 없어 맞춤 개발(GAMP Cat 5) 유지가 정당하다.
  • 무결성은 평문 SHA-256 해시체인(D-1)으로 구현하며, 이는 URS-041(변조 탐지)을 충족한다. 추가 강화(HMAC 등)는 필요 시 향후 변경관리로 재검토한다.

7. 기술 표준 (Standards)

  • 무결성 해시: SHA-256
  • 비밀번호: argon2(또는 bcrypt) salted hash
  • 시각: OS 시스템 시각 + 변경 탐지, (옵션) NTP
  • 키 보호: Windows DPAPI
  • 데이터: 임베디드 SQL DB + 표준 이미지 포맷(PNG/JPEG)

8. 설계 검토 결론 (Conclusion)

  • 기존 솔루션 평가(§6) 결과 적합한 기성 솔루션이 없어 맞춤 개발을 유지하며, 무결성은 평문 SHA-256 해시체인으로 구현한다.
  • 제안 설계는 URS 46개 요구사항을 충족하며, URS-002의 일탈 중 설계로 해결 가능한 부분(DEV-01 해시 체이닝, DEV-02 시간 탐지, DEV-03 키 관리)을 설계에 반영했다.
  • 환경·절차 의존 일탈(DEV-04~06)은 설계 범위 밖이며 IQ/SOP로 처리된다(URS-002 §4.3 체크리스트).
  • 따라서 본 설계는 적격하며 구현(빌드) 및 IQ 단계 진입이 가능하다.
