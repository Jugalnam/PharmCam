PharmCam Beta User Manual

PharmCam 사용자 매뉴얼

GMP 검체·시료 사진 기록 시스템의 베타 테스트용 사용 안내서입니다. 본 매뉴얼은 초기 타입 확정 후 기능 확인과 사용자 피드백 수집을 위해 작성되었습니다.

Version 0.1.0 Beta Windows Portable Controlled Printout

1. 베타 테스트 목적

PharmCam은 검체·시료 사진을 전자기록으로 저장하고, 사용자 식별·감사추적·원본 이미지 무결성을 함께 관리하는 Windows 데스크톱 앱입니다.

중요: 현재 빌드는 IQ/OQ/PQ 완료 전 베타 테스트용입니다. 실제 GMP 운영 기록으로 사용하기 전에는 회사 절차에 따른 검증과 승인이 필요합니다.

2. 실행과 로그인

  1. PharmCam-0.1.0-portable.exe를 실행합니다.
  2. 초기 관리자 계정으로 로그인합니다: admin / Admin123!
  3. 최초 로그인 시 비밀번호 변경 화면이 표시되면 새 비밀번호로 변경합니다.
역할주요 권한
operator촬영, 본인 촬영 기록 조회, 감사추적 조회
reviewer전체 기록 조회, 사용자별 기록 필터, 전자서명, 내보내기
admin설정, 계정관리, 환경 점검, 전체 기록 조회

3. 촬영

  1. 상단 메뉴에서 촬영 탭을 선택합니다.
  2. 시험번호와 시료 ID를 입력합니다. 회사별 추가 메타데이터 항목이 설정되어 있으면 함께 입력합니다.
  3. 카메라 미리보기를 확인한 뒤 촬영합니다.
  4. 촬영 이미지가 적절하지 않으면 재촬영합니다.
  5. 저장을 누르면 원본 이미지, 메타데이터, 작업자, 촬영일시가 함께 저장됩니다.

필수 항목이 비어 있으면 저장이 차단됩니다. 저장 성공/실패 및 촬영 행위는 감사추적에 기록됩니다.

4. 기록 확인

기록확인 탭은 기본적으로 오늘 날짜의 기록만 표시합니다.

  1. 상단 메뉴에서 기록 목록 탭을 선택합니다.
  2. 시작일/종료일을 선택해 기간별로 조회합니다.
  3. reviewer/admin은 작업자 필터로 특정 사용자의 기록만 볼 수 있습니다.
  4. operator는 본인이 촬영한 기록만 조회할 수 있습니다.
  5. 목록에서 기록을 선택하면 상세정보, 이미지 해시, 무결성 검증 결과를 확인할 수 있습니다.

5. 통제 인쇄

PharmCam 전자기록이 원본입니다. 종이 출력이 필요한 경우에는 PharmCam의 통제 인쇄 기능으로 출력한 문서만 공식 출력본으로 인정합니다.

  1. 기록 목록에서 출력할 기록을 선택합니다.
  2. 인쇄 미리보기를 눌러 공식 출력본 화면을 확인합니다.
  3. 미리보기에서 User ID, 시험번호, 촬영일시, 기록 ID, 파일명을 확인합니다.
  4. 공식 출력본 인쇄를 눌러 Windows 인쇄 대화상자에서 프린터를 선택합니다.
저장 폴더의 이미지 파일을 OS 또는 외부 프로그램으로 직접 인쇄한 출력물은 공식 출력본이 아닙니다. 인쇄 성공/실패는 감사추적에 기록됩니다.

6. 감사추적

  1. 감사추적 탭을 선택합니다.
  2. 사용자 드롭다운에서 admin (#1), leesi (#2) 같은 사용자명을 선택해 필터링합니다.
  3. 시작일/종료일 달력으로 조회 기간을 선택합니다.
  4. 행위(action) 필터로 login, capture, print, config_change 등을 좁혀 볼 수 있습니다.
  5. verifyChain 버튼으로 감사추적 해시체인 무결성을 확인합니다.

감사추적은 append-only입니다. 사용자가 수정하거나 삭제할 수 없습니다.

7. 계정 관리

관리자는 계정 관리 탭에서 사용자 생성과 비활성화를 수행할 수 있습니다.

  • 퇴사자 또는 사용 중지 계정은 비활성화합니다.
  • 비활성화된 계정은 로그인할 수 없습니다.
  • 권한 매트릭스는 읽기전용으로 표시됩니다.

8. 설정

관리자는 설정 탭에서 회사별 값을 조정할 수 있습니다.

설정 영역설명
세션/비밀번호 정책세션 타임아웃, 비밀번호 최소 길이, 로그인 실패 잠금 등을 설정합니다.
전자서명전자서명 사용 여부와 적용 행위를 설정합니다.
저장 위치로컬 저장 경로를 지정합니다. 네트워크/공유 폴더는 허용되지 않습니다.
촬영 메타데이터 항목회사별 추가 메타데이터 항목과 필수 여부를 설정합니다.

감사추적, 원본 무결성, 인증, 시각 출처 같은 GMP 코어 설정은 비활성화할 수 없습니다.

9. 환경 점검

관리자는 환경 점검 탭에서 IQ 준비용 자가점검을 실행할 수 있습니다.

  • safeStorage(DPAPI) 사용 가능 여부
  • 데이터 폴더 쓰기 권한
  • 감사추적 해시체인 무결성
  • 코어 설정 잠금 상태
  • 현재 설정 명세(configuration spec)

자가점검 결과는 JSON 파일로 내보낼 수 있습니다.

10. 베타 테스트 체크포인트

확인 항목기대 결과
촬영 저장사진과 메타데이터가 저장되고 기록 목록에 표시됨
operator 기록 조회본인 촬영 기록만 표시됨
reviewer/admin 기록 조회전체 기록과 작업자 필터 사용 가능
통제 인쇄미리보기와 실제 출력물에 필수 식별정보가 포함됨
감사추적사용자명 드롭다운, 날짜 필터, print/capture/login 기록 확인 가능
환경 점검자가점검 결과가 표시되고 JSON 내보내기 가능
