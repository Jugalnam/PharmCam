이 문서가 하는 일: "시스템이 반드시 해야 할 일"을 요구사항으로 정의한다.
각 요구사항은 RA의 위험 통제에서 도출되며(추적성), 이후 DQ(설계)·OQ/PQ(검증)로 연결된다.
PharmCam은 엄격한 GMP 코어 + 회사별 설정 유연성 원칙을 따른다(VP/RA 강의 참조).

1. 요구사항 분류 태그

태그의미
[필수]무결성 코어. 항상 적용되며 설정으로 끌 수 없음
[설정가능]회사별로 on/off 하거나 값을 조정할 수 있음
[조건부]특정 선택 모듈을 사용할 때만 필수가 됨
모든 요구사항은 출처 위험 ID(RA)를 명시한다. 추적성은 docs/traceability_matrix.md 참조.

2.5 DAT — 데이터 무결성 · 감사추적

URS ID태그요구사항출처(RA)
URS-040[필수]모든 행위(촬영·조회·수정·삭제·로그인·설정변경)에 감사추적(누가·언제·무엇을·이전/이후값) 기록. 행위자·대상 사용자는 감사추적 조회·출력 시 사람이 식별 가능한 형태(사용자명, 내부 ID 병기)로 표시되어야 한다(ALCOA+ Legible)DAT-02

URS-043[필수]원본 이미지는 무결성 검증값(해시)과 함께 읽기전용으로 보관해야 한다DAT-03
URS-044[필수]기록 일시는 신뢰할 수 있는 시각 출처를 사용하고, 시스템 시간 변경을 감지·기록DAT-04, INF-01

URS-046[필수]기록은 임의 수정·삭제 불가하며, 정정은 사유와 이력을 남기는 방식으로만 가능DAT-02
URS-047[설정가능]원본 이미지·데이터의 저장 위치(경로)를 지정·조회할 수 있어야 한다(로컬 경로로 한정 — 네트워크/공유 폴더 제외). 기본값을 제공하고, 변경은 권한자(관리자)만 가능하며 변경 이력(이전 경로→새 경로)을 감사추적에 남긴다. 위치 변경 시 기존 데이터는 자동 이전하지 않으며, 기록별 원본 경로가 보존되어 계속 접근·검증 가능해야 한다. 지정 위치에서도 무결성 보호(읽기전용·해시·감사추적)는 동일하게 강제된다(보호는 비활성화 불가). 검토·폴더감사·백업/복원 검증을 위해 현재 저장 위치를 화면에서 확인할 수 있어야 한다DAT-08 (신규)

URS ID태그요구사항출처(RA)
URS-090[필수]정기 검토를 위해 감사추적을 조회·필터·출력할 수 있어야 한다HUM-02

URS-092[설정가능]사용자 교육을 위한 도움말/가이드를 제공한다HUM-01
URS-093[필수]PharmCam에서 생성된 전자기록이 원본이며, 종이로 사용할 경우에는 PharmCam의 통제 인쇄(controlled printout) 기능으로 출력한 문서만 공식 출력본으로 인정한다. 인쇄 전 미리보기에는 최소한 User ID, 시험번호, 촬영일시, 기록 ID, 파일명이 표시되어야 하며, 사용자는 이를 확인한 뒤 인쇄해야 한다. 인쇄 행위는 누가·언제·어떤 기록을 인쇄했는지 감사추적에 기록되어야 한다. 저장 폴더의 이미지 파일을 OS 또는 외부 프로그램으로 직접 인쇄한 출력물은 PharmCam 통제·감사추적·메타데이터 확인을 거치지 않았으므로 공식 출력본으로 보지 않는다DAT-09
URS-094[필수]기록 목록은 날짜/기간으로 필터링할 수 있어야 하며, 기록확인 탭의 기본 조회 범위는 오늘 날짜로 제한되어야 한다. 사용자는 필요 시 허용된 권한 범위 내에서 과거 날짜 또는 기간을 선택해 조회할 수 있어야 한다DAT-10
URS-095[필수]기록 조회 범위는 역할 기반으로 통제되어야 한다. operator는 본인이 촬영한 기록만 조회할 수 있어야 하며, reviewer/admin은 전체 기록 조회와 사용자별 필터링이 가능해야 한다. 이 통제는 Renderer 표시 제어가 아니라 Main 프로세스의 조회 쿼리에서 강제되어야 한다DAT-10, AUTH-02
