URS-001 — User Requirement Specification (사용자 요구사항 명세)

시스템명: PharmCam — 검체·시료 사진 기록 시스템
문서 ID: URS-001
버전: 1.4 (Approved — 2026-06-09 확정: URS-053 설치·데이터 접근통제 신설)
작성일: 2026-06-05
확정일: 2026-06-05 (v1.0) · 2026-06-08 (v1.1·v1.2) · 2026-06-09 (v1.3·v1.4)
상위 문서: VP-001 (Validation Plan), RA-001 (Risk Assessment)

이 문서가 하는 일: "시스템이 반드시 해야 할 일"을 요구사항으로 정의한다.
각 요구사항은 RA의 위험 통제에서 도출되며(추적성), 이후 DQ(설계)·OQ/PQ(검증)로 연결된다.
PharmCam은 엄격한 GMP 코어 + 회사별 설정 유연성 원칙을 따른다(VP/RA 강의 참조).

1. 요구사항 분류 태그

태그 의미
[필수] 무결성 코어. 항상 적용되며 설정으로 끌 수 없음
[설정가능] 회사별로 on/off 하거나 값을 조정할 수 있음
[조건부] 특정 선택 모듈을 사용할 때만 필수가 됨

모든 요구사항은 출처 위험 ID(RA)를 명시한다. 추적성은 docs/traceability_matrix.md 참조.

URS-031 [설정가능] 필수 메타데이터 항목 구성(예: 검체ID·배치번호 추가)을 회사별 설정 가능 DAT-01

URS-034 [필수] 촬영 대상 시험정보를 화면에 표시해 오촬영을 예방해야 한다 SW-03

2.5 DAT — 데이터 무결성 · 감사추적

URS ID 태그 요구사항 출처(RA)
URS-040 [필수] 모든 행위(촬영·조회·수정·삭제·로그인·설정변경)에 감사추적(누가·언제·무엇을·이전/이후값) 기록. 행위자·대상 사용자는 감사추적 조회·출력 시 사람이 식별 가능한 형태(사용자명, 내부 ID 병기)로 표시되어야 한다(ALCOA+ Legible) DAT-02
URS-041 [필수] 감사추적은 append-only이며 사용자가 수정·삭제할 수 없어야 한다 DAT-02
URS-042 [필수] 감사추적 기능은 비활성화할 수 없어야 한다(설정 대상 아님) DAT-07
URS-043 [필수] 원본 이미지는 무결성 검증값(해시)과 함께 읽기전용으로 보관해야 한다 DAT-03
URS-044 [필수] 기록 일시는 신뢰할 수 있는 시각 출처를 사용하고, 시스템 시간 변경을 감지·기록 DAT-04, INF-01
URS-045 [필수] 메타데이터와 이미지를 묶어 저장하고, 중도 실패·재촬영도 기록해야 한다 DAT-05, DAT-06
URS-046 [필수] 기록은 임의 수정·삭제 불가하며, 정정은 사유와 이력을 남기는 방식으로만 가능 DAT-02
URS-047 [설정가능] 원본 이미지·데이터의 저장 위치(경로)를 지정·조회할 수 있어야 한다(로컬 경로로 한정 — 네트워크/공유 폴더 제외). 기본값을 제공하고, 변경은 권한자(관리자)만 가능하며 변경 이력(이전 경로→새 경로)을 감사추적에 남긴다. 위치 변경 시 기존 데이터는 자동 이전하지 않으며, 기록별 원본 경로가 보존되어 계속 접근·검증 가능해야 한다. 지정 위치에서도 무결성 보호(읽기전용·해시·감사추적)는 동일하게 강제된다(보호는 비활성화 불가). 검토·폴더감사·백업/복원 검증을 위해 현재 저장 위치를 화면에서 확인할 수 있어야 한다 DAT-08 (신규)

2.6 SEC — 보안 · 암호화

URS ID 태그 요구사항 출처(RA)
URS-050 [설정가능] 저장 데이터(이미지·메타데이터)를 암호화 보관(encryption at rest). 사용 여부 설정 SEC-01
URS-051 [필수] 비밀번호는 평문 저장·전송 금지, 안전한 해시로 보관해야 한다 SEC-03
URS-052 [설정가능] 단독 PC·외부연결 차단·악성코드 보호 등 환경 보안 요건 SEC-04
URS-053 [필수] 설치·데이터 접근통제: 프로그램은 권한자(관리자)만 변경 가능한 위치(예: Program Files, per-machine)에 설치하고, 작업자는 표준 Windows 사용자로 운영한다(로컬 관리자 금지). 데이터 폴더는 작업자의 직접 수정·삭제를 제한(ACL)한다. 동일 사용자 토큰 특성상 완전 예방이 불가한 잔여 위험은 암호화(at rest)·해시체인 탐지·정기 검토로 관리하며, 적용 상태는 IQ에서 확인한다 SEC-06

2.7 BAK — 백업 · 복구 · 보존

URS ID 태그 요구사항 출처(RA)
URS-060 [설정가능] 백업 방식 선택: 내장 백업 / 외부(회사 자체) 경로 지정 / 내장 백업 비활성 BAK-01
URS-061 [조건부] 내장 백업 사용 시 백업 무결성·복구 가능성을 검증할 수 있어야 한다 BAK-02
URS-062 [설정가능] 보존기간을 설정 가능하고, 보존기간 동안 판독 가능한 표준 포맷으로 유지 BAK-03
URS-063 [필수] 저장공간 임계치 도달 시 경고해야 한다 BAK-04

2.8 CFG — 설정 관리 (유연성 가드레일)

URS ID 태그 요구사항 출처(RA)
URS-070 [필수] 회사별 설정 프로필(configuration)을 가지며, 설정 가능 기능의 on/off·값을 정의한다 아키텍처 원칙
URS-071 [필수] 설정 변경은 권한자(관리자)만 가능하고, 모든 변경은 감사추적에 기록되어야 한다 신규 위험(설정 우회)
URS-072 [필수] 무결성 코어(감사추적·식별·원본 보호·신뢰 시각·필수 메타데이터)는 설정으로 비활성화 불가 아키텍처 원칙
URS-073 [필수] 현재 적용된 설정 상태를 조회·출력할 수 있어야 한다(configuration spec, 검증·감사용). 역할별 권한 매핑(RBAC 매트릭스)도 읽기전용으로 조회할 수 있어야 한다(권한 분리 검증·감사용) 회사별 IQ/OQ

URS-081 [조건부] 연동 사용 시 외부 전송 데이터의 무결성(정확한 전송·확인)을 보장해야 한다 DI

2.10 NFR — 운영 · 비기능

URS ID 태그 요구사항 출처(RA)
URS-090 [필수] 정기 검토를 위해 감사추적을 조회·필터·출력할 수 있어야 한다 HUM-02
URS-091 [필수] 미인가 반출 방지를 위한 내보내기 통제(권한 기반) INF-02
URS-092 [설정가능] 사용자 교육을 위한 도움말/가이드를 제공한다 HUM-01
URS-093 [필수] PharmCam에서 생성된 전자기록이 원본이며, 종이로 사용할 경우에는 PharmCam의 통제 인쇄(controlled printout) 기능으로 출력한 문서만 공식 출력본으로 인정한다. 인쇄 전 미리보기에는 최소한 User ID, 시험번호, 촬영일시, 기록 ID, 파일명이 표시되어야 하며, 사용자는 이를 확인한 뒤 인쇄해야 한다. 인쇄 행위는 누가·언제·어떤 기록을 인쇄했는지 감사추적에 기록되어야 한다. 저장 폴더의 이미지 파일을 OS 또는 외부 프로그램으로 직접 인쇄한 출력물은 PharmCam 통제·감사추적·메타데이터 확인을 거치지 않았으므로 공식 출력본으로 보지 않는다 DAT-09
URS-094 [필수] 기록 목록은 날짜/기간으로 필터링할 수 있어야 하며, 기록확인 탭의 기본 조회 범위는 오늘 날짜로 제한되어야 한다. 사용자는 필요 시 허용된 권한 범위 내에서 과거 날짜 또는 기간을 선택해 조회할 수 있어야 한다 DAT-10
URS-095 [필수] 기록 조회 범위는 역할 기반으로 통제되어야 한다. operator는 본인이 촬영한 기록만 조회할 수 있어야 하며, reviewer/admin은 전체 기록 조회와 사용자별 필터링이 가능해야 한다. 이 통제는 Renderer 표시 제어가 아니라 Main 프로세스의 조회 쿼리에서 강제되어야 한다 DAT-10, AUTH-02
