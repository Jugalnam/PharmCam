# URS-001 — User Requirement Specification (사용자 요구사항 명세)

**시스템명:** PharmCam — 검체·시료 사진 기록 시스템
**문서 ID:** URS-001
**버전:** 1.3 (Approved — 2026-06-09 확정: URS-035 촬영 카메라 선택/전환 신설)
**작성일:** 2026-06-05
**확정일:** 2026-06-05 (v1.0) · 2026-06-08 (v1.1·v1.2) · 2026-06-09 (v1.3)
**상위 문서:** VP-001 (Validation Plan), RA-001 (Risk Assessment)

> **이 문서가 하는 일:** "시스템이 반드시 해야 할 일"을 요구사항으로 정의한다.
> 각 요구사항은 RA의 위험 통제에서 도출되며(추적성), 이후 DQ(설계)·OQ/PQ(검증)로 연결된다.
> PharmCam은 **엄격한 GMP 코어 + 회사별 설정 유연성** 원칙을 따른다(VP/RA 강의 참조).

### 2.5 DAT — 데이터 무결성 · 감사추적

| URS ID | 태그 | 요구사항 | 출처(RA) |
|---|---|---|---|
| URS-040 | [필수] | 모든 행위(촬영·조회·수정·삭제·로그인·설정변경)에 감사추적(누가·언제·무엇을·이전/이후값) 기록. **행위자·대상 사용자는 감사추적 조회·출력 시 사람이 식별 가능한 형태(사용자명, 내부 ID 병기)로 표시**되어야 한다(ALCOA+ Legible) | DAT-02 |

| URS-042 | [필수] | 감사추적 기능은 비활성화할 수 없어야 한다(설정 대상 아님) | DAT-07 |

| URS-045 | [필수] | 메타데이터와 이미지를 묶어 저장하고, 중도 실패·재촬영도 기록해야 한다 | DAT-05, DAT-06 |

| URS-047 | [설정가능] | 원본 이미지·데이터의 **저장 위치(경로)를 지정·조회**할 수 있어야 한다(**로컬 경로로 한정** — 네트워크/공유 폴더 제외). 기본값을 제공하고, 변경은 권한자(관리자)만 가능하며 **변경 이력(이전 경로→새 경로)을 감사추적에 남긴다**. 위치 변경 시 기존 데이터는 자동 이전하지 않으며, 기록별 원본 경로가 보존되어 계속 접근·검증 가능해야 한다. **지정 위치에서도 무결성 보호(읽기전용·해시·감사추적)는 동일하게 강제**된다(보호는 비활성화 불가). 검토·폴더감사·백업/복원 검증을 위해 현재 저장 위치를 화면에서 확인할 수 있어야 한다 | DAT-08 (신규) |

### 2.10 NFR — 운영 · 비기능

| URS ID | 태그 | 요구사항 | 출처(RA) |
|---|---|---|---|
| URS-090 | [필수] | 정기 검토를 위해 감사추적을 조회·필터·출력할 수 있어야 한다 | HUM-02 |
| URS-091 | [필수] | 미인가 반출 방지를 위한 내보내기 통제(권한 기반) | INF-02 |

| URS-093 | [필수] | PharmCam에서 생성된 전자기록이 원본이며, 종이로 사용할 경우에는 PharmCam의 **통제 인쇄(controlled printout)** 기능으로 출력한 문서만 공식 출력본으로 인정한다. 인쇄 전 미리보기에는 최소한 **User ID, 시험번호, 촬영일시, 기록 ID, 파일명**이 표시되어야 하며, 사용자는 이를 확인한 뒤 인쇄해야 한다. 인쇄 행위는 누가·언제·어떤 기록을 인쇄했는지 감사추적에 기록되어야 한다. 저장 폴더의 이미지 파일을 OS 또는 외부 프로그램으로 직접 인쇄한 출력물은 PharmCam 통제·감사추적·메타데이터 확인을 거치지 않았으므로 공식 출력본으로 보지 않는다 | DAT-09 |
| URS-094 | [필수] | 기록 목록은 날짜/기간으로 필터링할 수 있어야 하며, 기록확인 탭의 기본 조회 범위는 **오늘 날짜**로 제한되어야 한다. 사용자는 필요 시 허용된 권한 범위 내에서 과거 날짜 또는 기간을 선택해 조회할 수 있어야 한다 | DAT-10 |
| URS-095 | [필수] | 기록 조회 범위는 역할 기반으로 통제되어야 한다. operator는 본인이 촬영한 기록만 조회할 수 있어야 하며, reviewer/admin은 전체 기록 조회와 사용자별 필터링이 가능해야 한다. 이 통제는 Renderer 표시 제어가 아니라 Main 프로세스의 조회 쿼리에서 강제되어야 한다 | DAT-10, AUTH-02 |
