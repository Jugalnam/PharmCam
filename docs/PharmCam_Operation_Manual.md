## 1. 목적 및 범위

본 매뉴얼은 PharmCam을 사용하는 **작업자(operator) · 검토자(reviewer) · 관리자(admin)** 가 검체·시료 사진을 GMP 요건에 맞게 기록·조회·통제하기 위한 표준 운영 절차를 규정한다.

- **대상 독자:** QC 시험자, QC 검토자, 시스템 관리자
- **적용 시스템:** PharmCam v0.1.0 (URS/DQ v1.4 기준)
- **적용 행위:** 검체 촬영, 전자기록 조회, 통제 인쇄, 감사추적 확인, 계정·설정 관리, 환경 점검

본 매뉴얼은 시스템 설치·검증 절차서(IQ/OQ/PQ)나 밸리데이션 계획서(VP)를 대체하지 않는다.

## 2. 시스템 개요

PharmCam은 검체·시료 사진을 **위·변조 없이 전자기록으로 저장**하고, 작업자 식별·감사추적·원본 무결성을 함께 관리하는 Windows 데스크톱 앱이다. 데이터 무결성 원칙 **ALCOA+** 를 코드 레벨에서 보장한다.

- **Attributable(귀속):** 로그인한 작업자 없이는 기록을 남기지 않는다.
- **Legible/Contemporaneous(가독·동시성):** 신뢰 시각으로 행위 즉시 기록한다.
- **Original/Accurate(원본·정확):** 원본 이미지를 SHA-256 해시로 봉인하고, 감사추적은 append-only(추가 전용)로 보관한다.

기능은 두 층으로 나뉜다. **끌 수 없는 무결성 코어**(감사추적·작업자 식별·원본 보호·신뢰 시각·필수 메타데이터)와 **회사별 설정 가능 모듈**(전자서명·백업·저장 위치·메타데이터 항목·비밀번호 정책 등)이다.

## 3. 용어

| 용어 | 설명 |
|---|---|
| GMP | Good Manufacturing Practice. 의약품 제조·품질관리 기준 |
| CSV | Computer System Validation. 컴퓨터 시스템 밸리데이션 |
| ALCOA+ | 데이터 무결성 9대 속성 (Attributable·Legible·Contemporaneous·Original·Accurate + Complete·Consistent·Enduring·Available) |
| 감사추적(audit trail) | 누가·언제·무엇을 했는지에 대한 변경 불가 기록 |
| append-only | 추가만 가능하고 수정·삭제가 불가능한 저장 방식 |
| 해시(SHA-256) | 데이터 변조 여부를 검출하는 디지털 지문 |
| 통제 인쇄 | 원본 해시 검증을 거쳐 공식 출력본을 인쇄하는 기능 |
| RBAC | Role-Based Access Control. 역할 기반 접근 통제 |

## 4. 역할과 권한 (RBAC)

| 역할 | 기록 조회 범위 | 주요 권한 |
|---|---|---|
| **operator** (작업자) | 본인 촬영 기록만 | 촬영, 본인 기록 조회, 통제 인쇄, 감사추적 조회, 도움말 |
| **reviewer** (검토자) | 전체 기록 + 작업자 필터 | 전체 기록 조회, 전자서명, 내보내기, 통제 인쇄, 감사추적 조회 |
| **admin** (관리자) | 전체 기록 + 작업자 필터 | 위 항목 + 계정 관리, 설정, 환경 점검 |

> 역할별로 상단 메뉴에 보이는 탭이 다르다. **계정 관리·설정·환경 점검** 탭은 관리자에게만 표시된다.

## 5. 설치 및 데이터 위치

### 5.1 설치 경로

정식 배포본은 **per-machine 설치**로, 기본 설치 경로는 `C:\Program Files\PharmCam\` 이다. 이 경로는 관리자 권한이 있어야 변경할 수 있어, 표준 사용자(작업자)가 설치된 프로그램 파일을 변조할 수 없다(무결성 다층 방어).

### 5.2 데이터 저장 위치

기록(DB·이미지·암호화 키·백업)은 기본적으로 사용자별 데이터 폴더에 저장된다.

| 데이터 | 기본 경로 |
|---|---|
| DB·이미지·키 | `%APPDATA%\pharmcam\data\` |
| 백업 | `%APPDATA%\pharmcam\backups\` |

관리자는 **설정 → 저장 위치**에서 로컬 경로를 변경할 수 있다(네트워크/공유 폴더는 허용되지 않음). 실제 적용 경로는 **환경 점검** 탭에서 확인한다.

### 5.3 백업·보존

- 데이터 폴더는 회사 백업 정책에 따라 정기 백업한다.
- 원본 이미지와 감사추적은 보존기간 동안 변경 없이 보관한다(Enduring/Available).
- 저장 매체의 가용 공간을 주기적으로 점검한다(촬영 화면에 저장공간 부족 경고가 표시됨).

## 6. 시작하기 — 실행과 로그인

1. 바탕화면 또는 시작 메뉴에서 **PharmCam**을 실행한다.
2. 초기 관리자 계정으로 로그인한다: `admin / Admin123!`
3. 최초 로그인 시 비밀번호 변경 화면이 표시되면 회사 비밀번호 정책에 맞는 새 비밀번호로 변경한다.
4. 관리자는 이후 **계정 관리** 탭에서 실제 작업자·검토자 계정을 생성한다.

> **보안:** 초기 비밀번호(`Admin123!`)는 최초 로그인 시 반드시 변경한다. 비밀번호는 공유하지 않으며, 로그인 계정은 곧 기록의 귀속(Attributable) 근거다.

## 7. 촬영 탭 (전체)

검체를 촬영하고 메타데이터와 함께 전자기록으로 저장한다.

1. 상단 메뉴에서 **촬영** 탭을 선택한다.
2. **시험번호·시료 ID** 등 메타데이터를 입력한다. 회사별 추가 항목이 설정되어 있으면 함께 입력한다.
3. 카메라가 여러 대 연결된 경우 **카메라 선택/전환** 버튼으로 사용할 장치를 고른다(전·후면, 외장 웹캠 등).
4. 미리보기를 확인한 뒤 촬영한다. 결과가 적절하지 않으면 **재촬영**한다.
5. **저장**을 누르면 원본 이미지, 메타데이터, 작업자, 촬영일시가 함께 저장되고 원본은 SHA-256 해시로 봉인된다.

> **참고:** 필수 항목이 비어 있으면 저장이 차단된다. 저장공간이 부족하면 사전 경고가 표시된다. 촬영·저장 성공/실패는 모두 감사추적에 기록된다.

## 8. 기록 목록 탭 (전체)

저장된 기록을 조회하고 상세·해시·무결성을 확인하며, 통제 인쇄를 수행한다. 기본적으로 **오늘 날짜**의 기록만 표시된다.

1. **기록 목록** 탭을 선택한다.
2. 시작일/종료일을 선택해 기간별로 조회한다.
3. operator는 **본인 촬영 기록만**, reviewer/admin은 전체 기록을 보며 **작업자 필터**로 특정 사용자의 기록만 볼 수 있다.
4. 목록에서 기록을 선택하면 상세정보, 이미지, 해시값, 무결성 검증 결과, 커스텀 메타데이터를 확인한다.

### 8.1 통제 인쇄

PharmCam 전자기록이 원본이다. 종이 출력이 필요하면 **통제 인쇄**로 출력한 문서만 공식 출력본으로 인정한다.

1. 기록 목록에서 출력할 기록을 선택한다.
2. **인쇄 미리보기**로 공식 출력본 화면(User ID·시험번호·촬영일시·기록 ID·파일명 포함)을 확인한다.
3. **공식 출력본 인쇄**를 눌러 프린터를 선택해 출력한다.

> **경고:** 저장 폴더의 이미지 파일을 OS나 외부 프로그램으로 직접 인쇄한 출력물은 공식 출력본이 아니다. 인쇄 전 원본 해시가 검증되며, 인쇄 성공/실패는 감사추적에 기록된다.

## 9. 감사추적 탭 (전체)

누가·언제·무엇을 했는지에 대한 변경 불가 기록을 조회·검증한다.

1. **감사추적** 탭을 선택한다.
2. 사용자 드롭다운에서 `admin (#1)` 같은 사용자명을 선택해 필터링한다.
3. 시작일/종료일로 조회 기간을 선택한다.
4. 행위(action) 필터로 `login·capture·print·config_change` 등을 좁혀 본다.
5. **verifyChain** 버튼으로 감사추적 해시체인 무결성을 검증한다.
6. 필요 시 **CSV/PDF**로 내보낸다.

> **참고:** 감사추적은 append-only다. 사용자(관리자 포함)가 수정·삭제할 수 없으며, DB 트리거로도 변경이 차단된다. 한 줄이라도 변조되면 verifyChain에서 즉시 탐지된다.

## 10. 계정 관리 탭 (관리자)

작업자 식별의 출발점인 계정과 권한을 관리한다.

- **계정 생성/비활성화:** 신규 사용자를 생성하고, 퇴사자·사용 중지 계정은 **삭제 대신 비활성화**한다(과거 기록의 귀속 보존). 비활성화된 계정은 로그인할 수 없다.
- **권한 매트릭스:** 역할별 권한을 읽기전용으로 확인한다.
- 모든 계정 변경은 감사추적에 기록된다.

## 11. 설정 탭 (관리자)

회사별로 값을 조정한다. 무결성 코어 설정은 잠겨 있어 변경할 수 없다.

| 설정 영역 | 설명 |
|---|---|
| 세션/비밀번호 정책 | 세션 타임아웃, 비밀번호 최소 길이, 로그인 실패 잠금 등 |
| 전자서명 | 전자서명 사용 여부와 적용 행위 |
| 백업 | 백업 방식·경로 |
| 저장 위치 | 로컬 저장 경로 지정(네트워크/공유 폴더 불가) |
| 촬영 메타데이터 항목 | 회사별 추가 항목과 필수 여부 |
| 보존기간 | 기록 보존 기간 |

> **참고:** 감사추적·원본 무결성·인증·시각 출처 같은 GMP 코어 설정(`core_locked`)은 비활성화할 수 없다. 모든 설정 변경은 감사추적에 기록된다(누가 무엇을 켜고 껐는지).

## 12. 환경 점검 탭 (관리자)

설치·환경이 정상인지 자가점검한다(IQ 자가진단).

- safeStorage(DPAPI) 사용 가능 여부
- 데이터 폴더 쓰기 권한
- 감사추적 해시체인 무결성(verifyChain)
- 코어 설정 잠금 상태
- 현재 설정 명세(configuration spec)와 데이터 저장 폴더 경로

자가점검 결과는 JSON 파일로 내보낼 수 있다. 정기 점검 및 변경 후 점검의 증빙으로 보관한다.

## 13. 도움말 탭 (전체)

앱 내에서 각 기능의 사용법을 즉시 확인한다. 전 역할이 접근할 수 있어, 별도 매뉴얼 없이 현장에서 참고할 수 있다.

## 14. 운영 점검 체크리스트

| 주기 | 점검 항목 | 기대 결과 |
|---|---|---|
| 매일(시작 시) | 로그인·시각 표시 | 정상 로그인, 신뢰 시각 표시 |
| 매일 | 촬영 저장 | 사진·메타데이터 저장 및 기록 목록 표시 |
| 매일 | 저장공간 | 경고 없음(여유 공간 충분) |
| 주간 | verifyChain | 해시체인 무결성 정상(변조 없음) |
| 주간 | 환경 점검 | 모든 항목 정상, JSON 증빙 보관 |
| 변경 시 | 설정·계정 변경 | 감사추적에 변경 기록 확인 |
| 정기 | 데이터 백업 | 회사 정책에 따른 백업 완료 |

## 15. 문제 해결 (Troubleshooting)

| 증상 | 확인·조치 |
|---|---|
| 로그인 불가 | 계정 비활성화 여부 확인(관리자), 비밀번호 정책·실패 잠금 확인 |
| 카메라가 보이지 않음 | 카메라 연결·권한 확인, 카메라 선택/전환으로 장치 재선택 |
| 저장 차단 | 필수 메타데이터 항목 입력 여부 확인 |
| 저장공간 경고 | 데이터 폴더 디스크 여유 공간 확보 후 재시도 |
| 통제 인쇄 안 됨 | 프린터 연결 확인, 원본 해시 검증 실패 시 관리자에 보고 |
| verifyChain 실패 | **즉시 사용 중단 후 관리자·QA에 보고**(무결성 이상 가능성) |
| 기록이 보이지 않음 | 조회 날짜 범위·작업자 필터 확인(operator는 본인 기록만 표시) |

> **데이터 무결성 이상(verifyChain 실패)은 중대 사안이다.** 임의로 조작하지 말고 즉시 QA에 보고하여 일탈(deviation) 절차에 따른다.

## 16. 데이터 무결성·규정 준수 안내

- PharmCam의 전자기록이 **원본(raw data)**이며, 종이 출력본은 통제 인쇄본만 인정한다.
- 감사추적·원본 해시는 데이터 무결성의 핵심 증빙이다. 임의 삭제·변경은 불가하며 시도 자체가 기록된다.
- 시스템·설정 변경, 계정 변경은 변경 관리 절차에 따르고 필요 시 재검증(IQ/OQ)한다.
- 본 시스템을 GMP 운영에 사용하기 전 **IQ/OQ/PQ 완료 및 QA 승인**이 선행되어야 한다.
