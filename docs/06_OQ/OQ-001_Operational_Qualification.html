OQ-001 — Operational Qualification Test Protocol (운전 적격성 시험서)

시스템명: PharmCam — 검체·시료 사진 기록 시스템
문서 ID: OQ-001
버전: 0.2 (Draft — v1.4 범위로 확장: 카메라 선택·통제 인쇄·기록 조회 통제·접근통제)
작성일: 2026-06-08
상위 문서: VP-001, RA-001 v1.3, URS-001 v1.4, DQ-001 v1.4
추적성: docs/traceability_matrix.md

이 문서가 하는 일: PharmCam의 주요 기능이 DQ에서 정의한 설계대로 동작하는지 확인하기 위한 OQ 시험 항목을 정의한다.
본 문서는 초안이며, 실제 실행 전 QA/System Owner 검토와 승인 후 시험한다.

1. 목적 (Purpose)

PharmCam이 사전 정의된 기능 요구사항(URS)과 설계(DQ)에 따라 동작하는지 확인한다. 특히 데이터 무결성(ALCOA+), 감사추적, 접근통제, 원본 보호, 설정 관리, 저장 위치, 메타데이터 구성 기능을 위험 기반으로 시험한다.

2. 범위 (Scope)

2.1 포함

  • 로그인, 세션, 계정관리, RBAC
  • 촬영, 미리보기, 재촬영, 저장, 필수 메타데이터 검증
  • 원본 이미지 해시 및 읽기전용 보호
  • 감사추적 조회, 해시체인 검증, CSV/PDF 내보내기
  • 전자서명 기능
  • 설정 관리, 코어 잠금, 설정 명세 조회, 권한 매트릭스 조회
  • 저장 위치 지정 및 검증
  • 백업, 암호화 상태, LIMS 커넥터 스텁
  • 시스템 자가점검
  • 촬영 카메라 선택/전환
  • 통제 인쇄(controlled printout)
  • 기록 조회 통제(기본 오늘·날짜 필터·역할별 범위)
  • 접근통제(앱 레벨 operator 본인 기록 한정; 설치·OS 레벨은 IQ에서 확인)

2.2 제외

  • 실제 운영 PC 설치 적격성(IQ)
  • 실제 운영 시나리오 반복 수행(PQ)
  • 외부 LIMS 실연동
  • 카메라 장치 자체의 계측 검교정

3. 시험 전제조건

항목 기준
앱 버전 PharmCam 0.1.0 테스트 빌드
실행 환경 Windows 10/11 단독 PC 또는 검증용 개발 PC
초기 관리자 admin / Admin123! 후 비밀번호 변경
데이터 상태 신규 또는 시험용 DB 사용
네트워크 외부 네트워크 호출 없음
시험 증거 화면 캡처, 감사추적 출력, 내보낸 CSV/PDF, 자가점검 JSON

4. 공통 합격 기준

  • 실제 결과가 각 시험의 기대결과와 일치한다.
  • 실패, 거부, 권한 부족 같은 부정 시험은 사용자에게 명확한 메시지로 표시된다.
  • 데이터 변경 행위는 감사추적에 남는다.
  • 감사추적 해시체인 검증이 정상으로 표시된다.
  • 미결 중대 일탈이 있으면 OQ를 합격 처리하지 않는다.

5. OQ 시험 항목

Test ID 시험명 연결 URS 위험/설계 시험 절차 기대결과
OQ-TC-001 앱 실행 및 기본 화면 확인 URS-001, URS-002, URS-003 VP, 설계 전반 테스트 빌드를 실행하고 로그인 화면과 앱 제목을 확인한다. Windows 데스크톱 앱으로 실행되며 PharmCam 로그인 화면이 표시된다.
OQ-TC-002 로그인 및 개인 계정 강제 URS-010, URS-011, URS-051 AUTH-01, AUTH-03, D-5 로그인 전 촬영 화면 접근을 시도한 뒤, 유효 계정으로 로그인한다. 로그인 전 촬영·저장이 불가하고, 로그인 후 사용자 식별 상태가 표시된다.
OQ-TC-003 최초 비밀번호 변경 및 비밀번호 정책 URS-015, URS-051 SEC-02, SEC-03, D-5 초기 관리자 로그인 후 약한 비밀번호와 유효 비밀번호 변경을 각각 시도한다. 약한 비밀번호는 거부되고, 유효 비밀번호 변경 후 재로그인이 가능하다.
OQ-TC-004 로그인 실패 잠금 URS-016 SEC-05, D-5 설정된 실패 횟수 이상 잘못된 비밀번호로 로그인한다. 계정이 잠기거나 비활성화되며 감사추적에 실패 이력이 남는다.
OQ-TC-005 세션 타임아웃 URS-013 AUTH-04, D-5 세션 타임아웃 값을 짧게 설정하고 무활동 상태로 둔다. 설정 시간 후 세션이 만료되고 보호 기능 접근이 거부된다.
OQ-TC-006 계정 수명주기 관리 URS-014 AUTH-06, D-5 관리자로 사용자 생성 및 비활성화를 수행한다. 생성/비활성화가 반영되고 비활성 계정은 로그인할 수 없다.
OQ-TC-007 RBAC 및 권한 매트릭스 조회 URS-012, URS-073, URS-091 AUTH-02, INF-02, D-5, D-7 역할별 계정으로 로그인해 관리자 기능 접근과 권한 매트릭스 표시를 확인한다. 역할별 허용 기능만 접근 가능하며 권한 매트릭스는 읽기전용으로 표시된다.
OQ-TC-008 촬영, 미리보기, 재촬영 URS-030, URS-032, URS-034 DAT-01, SW-01, SW-03, D-9 카메라 미리보기 후 촬영, 재촬영, 다시 촬영을 수행한다. 미리보기와 재촬영이 정상 동작하고 저장 전 시험정보가 화면에 표시된다.
OQ-TC-009 필수 메타데이터 검증 및 저장 성공 확인 URS-030, URS-031, URS-033 DAT-01, SW-02, D-7, D-9 필수 메타데이터를 비운 저장과 모두 입력한 저장을 각각 시도한다. 미입력 저장은 차단되고, 정상 저장 시 성공 메시지와 기록 목록 반영이 확인된다.
OQ-TC-010 원본 이미지 보호 및 해시 검증 URS-043 DAT-03, DEV-01, D-2 저장된 기록을 조회하고 원본 파일 해시/읽기전용 상태를 확인한다. 원본 파일은 해시와 연결되고 읽기전용으로 보호된다.
OQ-TC-011 묶음 저장, 실패/재촬영 감사추적 URS-045 DAT-05, DAT-06, D-1, D-9 재촬영 후 저장하고 감사추적을 조회한다. 재촬영 및 저장 행위가 감사추적에 남고 이미지와 메타데이터가 한 기록으로 묶인다.
OQ-TC-012 정정 방식 및 임의 삭제 불가 URS-046 DAT-02, D-1 기존 기록의 정정 기능을 사용하고 임의 삭제 UI/API가 없는지 확인한다. 원본은 유지되고 정정 기록이 새 이력으로 생성된다. 삭제 기능은 제공되지 않는다.
OQ-TC-013 감사추적 append-only 및 해시체인 검증 URS-040, URS-041, URS-042 DAT-02, DAT-07, DEV-01, D-1 감사추적 조회, 해시체인 검증, 코어 설정 변경 시도를 수행한다. 감사추적은 조회만 가능하고 해시체인은 정상이며 감사추적 비활성화는 거부된다.
OQ-TC-014 감사추적 가독 표시 및 내보내기 URS-040, URS-090 HUM-02, D-1 감사추적 화면, CSV, PDF 출력에서 사용자 표시를 확인한다. 사용자와 대상이 username (#id)처럼 사람이 식별 가능한 형태로 표시된다.
OQ-TC-015 전자서명 URS-020, URS-021, URS-022, URS-023 AUTH-05, D-6 전자서명 설정 후 기록 서명 시 비밀번호 재인증을 수행한다. 서명자, 의미, 일시, 대상 기록 연결이 생성되고 재인증 실패 시 서명되지 않는다.
OQ-TC-016 설정 관리 및 코어 잠금 URS-070, URS-071, URS-072, URS-073 설정우회, D-7 관리자/비관리자 계정으로 설정 변경을 시도하고 설정 명세를 조회한다. 비관리자는 변경 불가, 관리자의 변경은 감사추적에 남고 core_locked 항목은 변경 불가하다.
OQ-TC-017 저장 위치 지정 및 로컬 경로 검증 URS-047 DAT-08, D-11 저장 위치를 로컬 경로로 변경하고 UNC/네트워크 경로를 시도한다. 로컬 쓰기 가능 경로만 허용되고 변경 전후 경로가 감사추적에 남는다. 기존 기록 경로는 유지된다.
OQ-TC-018 백업, 암호화, LIMS 선택 모듈 상태 URS-050, URS-060, URS-061, URS-062, URS-063, URS-080, URS-081 SEC-01, BAK, INT, D-4, D-8, D-10 설정 화면과 모듈 상태 화면에서 각 선택 모듈 상태와 동작을 확인한다. 선택 모듈 상태가 조회되고, 비활성 모듈은 명확히 비활성으로 표시된다. 내장 백업 사용 시 무결성 검증 결과가 제공된다.
OQ-TC-019 시간 출처 및 시간 이상 감지 URS-044 DAT-04, INF-01, DEV-02, D-3 기록 생성 시각이 단일 출처로 기록되는지 확인하고 시간 이상 감지 로그를 검토한다. 기록/감사추적 시각이 일관되며 시간 이상은 감사추적에 기록된다.
OQ-TC-020 시스템 자가점검 URS-052, URS-073 SEC-04, 회사별 IQ, D-7 관리자 환경 점검을 실행하고 결과 JSON을 내보낸다. safeStorage, 쓰기권한, 해시체인, 코어잠금, 설정 명세 결과가 표시되고 내보내진다.
OQ-TC-021 도움말/가이드 확인 URS-092 HUM-01 도움말 또는 사용자 가이드 접근 경로를 확인한다. 사용자가 기본 사용 흐름과 GMP 주의사항을 확인할 수 있다. 미구현 시 일탈로 기록한다.
OQ-TC-022 촬영 카메라 선택/전환 URS-035 SW-03, D-14 여러 카메라가 연결된 환경에서 카메라 선택 드롭다운과 "카메라 전환" 버튼을 사용한다. 연결된 카메라가 열거되고 선택·전환 시 미리보기가 해당 카메라로 바뀐다. 카메라가 1개면 전환 버튼은 비활성이다.
OQ-TC-023 통제 인쇄(controlled printout) URS-093 DAT-09, D-12 기록 상세에서 인쇄 미리보기를 열고 통제 인쇄를 수행한다. 미리보기에 User ID·시험번호·촬영일시·기록 ID·파일명이 표시되고, 인쇄 시 print 감사추적이 남는다(취소/실패는 print_failed). 저장 폴더 직접 인쇄는 공식 출력본이 아님이 문구로 표시된다.
OQ-TC-024 기록 조회 통제(기본 오늘·날짜·역할 범위) URS-094, URS-095, URS-053 DAT-10, AUTH-02, D-13, D-15 operator/reviewer/admin 계정으로 기록 목록을 조회하고 날짜·작업자 필터를 사용한다. 기본 조회는 오늘 날짜이며, operator는 본인 기록만(Main 강제), reviewer/admin은 전체 조회와 작업자 필터가 가능하다.
OQ-TC-025 메타데이터 항목 구성 및 기록 표시 URS-031 DAT-01, D-7, D-9 설정에서 추가 메타데이터 항목(예: 배치번호)을 정의해 촬영·저장하고 기록 상세를 조회한다. 촬영 화면에 추가 입력란이 표시되고, 필수 항목 미입력은 차단되며, 저장된 값이 기록 상세에 표시된다.

6. 일탈 처리

시험 결과가 기대결과와 다르면 다음 항목을 기록한다.

항목 내용
Deviation ID DEV-OQ-xxx
관련 Test ID 예: OQ-TC-017
현상 실제 결과와 기대결과 차이
영향 평가 데이터 무결성, 품질기록, 사용자 안전 영향
원인 코드 결함, 설정 오류, 절차 미비 등
조치 수정, 문서 보완, 재시험
종료 기준 조치 완료 및 재시험 합격
