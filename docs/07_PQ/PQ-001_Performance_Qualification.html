PQ-001 — Performance Qualification Protocol (성능 적격성 평가)

시스템명: PharmCam — 검체·시료 사진 기록 시스템
문서 ID: PQ-001
버전: 0.1 (Draft)
작성일: 2026-06-09
상위 문서: VP-001, URS-001 v1.4 (PQ는 URS와 짝을 이룬다), IQ-001, OQ-001
추적성: docs/traceability_matrix.md 및 본 문서 부록

이 문서가 하는 일: PharmCam이 실제 운영 환경에서 실제 사용자가 일상적으로 사용할 때, 사용자 요구사항(URS)을 일관되게·의도대로 충족하는지 확인하기 위한 PQ(성능 적격성) 시험 항목을 정의한다.
OQ가 "기능이 설계대로 작동하는가"를 본다면, PQ는 "실 환경·실 사용자·누적 사용 속에서 의도된 사용에 적합한가"를 본다.
본 문서는 초안이며, 실제 실행 전 QA/System Owner 검토·승인 후 파일럿 실사용으로 수행한다.

1. 목적 (Purpose)

IQ(설치)·OQ(기능)가 완료된 PharmCam을 실제 운영 PC·실제 QC 작업자·실제(또는 대표) 검체 워크플로우로 일정 기간 운용하여, 데이터 무결성(ALCOA+)·촬영 기록·검토/서명·통제 인쇄·백업/복구 등 의도된 사용이 안정적으로 충족됨을 입증한다. PQ 합격은 실운영 전환(go-live)의 근거가 된다.

2. 범위 (Scope)

2.1 포함 (실 환경·실사용·누적 시간이 변수인 항목)

  • 실사용자 일상 촬영 워크플로우(검체→메타데이터→저장)
  • 실제 조명·카메라·검체 조건에서의 이미지 적합성(판독·식별 가능성)
  • 다중 역할 실제 운영(작업자 본인기록 한정 / 검토자 검토·전자서명 / 관리자 운영)
  • 통제 인쇄본의 실제 기록 흐름(종이 배치기록 첨부 등) 적합성
  • 누적 운영에서의 데이터 무결성·감사추적·해시체인·원본 보존
  • 신뢰 시각·동시성(실 근무·교대)
  • 백업·복구 드릴(실데이터)
  • 저장 용량·보존·저장공간 경고의 실제 동작
  • 접근통제 및 SOP 준수(표준 사용자 운영, 정기 무결성 검토)
  • 지속 운영 신뢰성·성능

2.2 제외

  • 단위 기능의 작동/차단 확인(OQ-001에서 수행)
  • 설치 적격성(IQ-001에서 수행)
  • 카메라 장치 자체의 계측 검교정(장비 관리 SOP 영역)

3. 전제조건 (Prerequisites)

항목 기준
IQ/OQ IQ-001·OQ-001 승인·합격 완료(부적합 일탈 종료)
환경 실제 운영 PC(또는 동등 검증 PC), 폐쇄망
사용자 실제 역할별 계정(작업자·검토자·관리자), 표준 Windows 사용자
검체 실제 또는 대표 검체/시료(품질 영향 없는 시험 대상 권장)
SOP 촬영·검토·전자서명·통제 인쇄·백업·정기검토 SOP 확정
기간 파일럿 기간 사전 정의(권장 2~4주) 또는 누적 최소 기록 건수(예: ≥ 50건) — 승인 시 확정
설정 회사별 설정 명세(configuration spec) 확정·기록

4. PQ 수행 방식 (Approach)

  • 파일럿 기간 동안 실제 작업자가 평소 절차대로 PharmCam으로 검체를 촬영·기록한다.
  • 검토자/관리자는 실제 검토·서명·인쇄·백업·정기 무결성 검토를 수행한다.
  • 기간 종료(또는 목표 건수 도달) 시 누적 데이터·감사추적·백업·복구 결과를 평가한다.
  • 각 PQ-TC는 사전 합격 기준 충족 여부로 판정한다(적합/부적합/N/A). 부적합은 §7 일탈로 처리한다.

5. PQ 시험 항목 (PQ Test Cases)

PQ-TC 항목 연결 URS 실사용 시험 방법 합격 기준
PQ-TC-001 일상 촬영 워크플로우 URS-002, 030, 032, 034 작업자가 파일럿 기간 동안 실제 검체를 SOP대로 촬영·메타데이터 입력·저장 정해진 기간/건수 동안 누락·차질 없이 기록이 생성되고 시험정보·필수항목이 정확히 귀속된다
PQ-TC-002 이미지 적합성(현장 조건) URS-002, 035 실제 실험실 조명·카메라·검체로 촬영, 필요 시 카메라 선택/전환 저장된 이미지가 검체 식별·판독에 충분하며, 사용 카메라가 현장 하드웨어에서 정상 동작한다
PQ-TC-003 다중 역할 실제 운영 URS-012, 094, 095, 020~023 작업자·검토자·관리자가 각자 역할로 운영(본인기록 한정/검토·서명/계정·설정 관리) operator는 본인 기록만 조회·인쇄, reviewer/admin은 전체 조회·검토·서명이 흐름대로 수행된다
PQ-TC-004 통제 인쇄의 기록 흐름 적합성 URS-093 검토자가 통제 인쇄본을 출력해 실제 종이 기록(배치기록 등)에 첨부하는 절차 수행 통제 인쇄본에 식별정보가 표시되고 인쇄가 감사추적되며, 실제 문서 흐름에 적합하다
PQ-TC-005 누적 데이터 무결성·ALCOA+ URS-040~046 파일럿 누적 기록에 대해 감사추적 조회·해시체인 검증·원본 보호 상태 점검 누적량/기간에도 감사추적이 완전·불변(해시체인 통과)하고 원본·귀속·시각이 보존된다
PQ-TC-006 신뢰 시각·동시성 URS-044 실 근무/교대에 걸쳐 기록·감사추적 시각 일관성, 시간 이상 감지 로그 검토 기록 시각이 신뢰 출처로 일관되며, 시간 변경/이상은 감사추적에 남는다
PQ-TC-007 백업·복구 드릴(실데이터) URS-060, 061 누적 실데이터로 백업 수행 후 별도 환경에 복구·무결성 검증 백업이 생성되고 복구본의 무결성(해시) 검증을 통과한다(복구 가능성 입증)
PQ-TC-008 저장 용량·보존·공간 경고 URS-047, 062, 063 누적 저장 경과 관찰, 저장공간 경고 임계치·보존 동작 확인 지정 저장 위치에 안정 저장되고, 공간 임계치 경고가 적시 표시되며 보존 정책이 유지된다
PQ-TC-009 접근통제·SOP 준수 URS-053, HUM-02 표준 사용자 운영, 정기 verifyChain 검토 등 절차 준수 관찰 작업자는 표준 사용자로 운영되고 정기 무결성 검토가 SOP대로 수행·기록된다
PQ-TC-010 지속 운영 신뢰성·성능 운영(NFR) 파일럿 기간 일상 사용에서 안정성·응답성 관찰 중단·데이터 손실 없이 운용되며 일상 작업에 무리 없는 응답성을 보인다

6. 종합 합격 기준 (Acceptance Criteria)

  • PQ-TC 전 항목이 적합(Pass) 또는 정당한 N/A.
  • 파일럿 기간/목표 건수 동안 감사추적 해시체인 무결성 100% 유지, 데이터 손실 0건.
  • 발생한 일탈은 §7로 처리하고 조치·재확인 후 합격해야 실운영 전환을 승인한다.
  • 누적 기록·감사추적·백업/복구 결과·정기검토 기록을 PQ 보고서에 첨부한다.

7. 일탈 처리 (Deviation)

일탈 ID 관련 PQ-TC 현상 영향 평가(무결성/품질/안전) 원인 조치 종료 기준
DEV-PQ-xxx 조치 완료 및 재확인 합격
